# Skill: formal-writing

日本語のフォーマルな文書・連絡文（お礼・依頼・案内・お詫び・欠席連絡・挨拶文など）を作る。

## 原則

- 文中の個人情報（氏名・住所・事情）はこの会話の作文にだけ使う
- **事実は盛らない**。書かれていない美談・理由を勝手に創作しない
  （必要な情報が欠けていたら placeholder `〔◯◯〕` で残して指摘する）

## Step 1. 場面の特定（足りない分だけ 1 回で聞く）

| 項目           | 例                                                                |
| -------------- | ----------------------------------------------------------------- |
| 種類           | お礼 / 依頼 / 案内 / お詫び / 断り / 欠席連絡 / 挨拶 / 要望・苦情 |
| 宛先と関係     | 学校の先生 / 上司 / 取引先 / 近所 / 親戚 / 施設・行政             |
| 媒体           | メール / LINE等チャット / 手紙・はがき / 提出書類 / 掲示文        |
| 伝えるべき事実 | 日付・出来事・金額・希望など、文に入れる材料                      |
| トーン         | きっちり / 温かみ / 事務的、の希望があれば                        |

## Step 2. 敬語レベルと型を決める

### 敬語レベルの目安

| 宛先                         | レベル                                           |
| ---------------------------- | ------------------------------------------------ |
| 行政・施設・かしこまった相手 | 完全な敬体 + 頭語結語（手紙なら拝啓/敬具）       |
| 学校・園・職場               | 丁寧な敬体。過剰な謙譲は避け、簡潔さを優先       |
| 近所・保護者仲間・親戚       | 丁寧だが距離を作りすぎない敬体                   |
| チャット媒体                 | 敬体を保ちつつ、手紙的な定型（拝啓等）は使わない |

### 文書の基本形

1. 宛名 / 挨拶（媒体・季節に応じて。チャットでは省略か一言）
2. 名乗り（相手が自分を特定できる情報: 「◯年◯組◯◯の母」等）
3. 用件の核（お礼・依頼・報告を先に明確に）
4. 詳細・事情（必要最小限）
5. 結び（相手への配慮 + 今後の関係）
6. 署名・日付（媒体に応じて）

## Step 3. 書く

### 質の基準

- **用件が 3 行以内に伝わる**こと。前置きで本題を埋もれさせない
- 二重敬語（「おっしゃられる」）・バイト敬語（「〜のほう」「〜になります」）を使わない
- 季節の挨拶は手紙・かしこまったメールのみ。日付と季節を整合させる（7月に「新緑の候」を書かない)
- 依頼・要望・苦情は **相手が動きやすい形**にする:
  具体的に何をしてほしいか 1 点に絞る / 期限があれば明記 / 相手の事情への理解を一文添える
- お詫びは言い訳を先に書かない（謝意 → 事実 → 対応 → 再発防止の順）
- 欠席・断りは理由を詳細に書きすぎない（簡潔な理由 + お詫び + 代替があれば提示）

### 出力形式

- 本文をそのままコピペできる形で出す（コードブロックに入れる）
- ユーザーが埋める箇所は `〔氏名〕` `〔日付〕` で明示する
- 迷いそうな箇所（敬語の強さ・入れるか迷う一文）は本文の後に「調整ポイント」として 2〜3 個添える

## Step 4. バリエーション対応

- 「もう少し柔らかく / 硬く」→ 敬語レベルを 1 段動かして全文を再掲する
- 長さの指定（「3 行で」「はがきに収まる量で」）に合わせて要素を削る優先順位:
  季節の挨拶 → 詳細事情 → 結びの装飾、の順に削る。**用件と名乗りは最後まで残す**

## やってはいけないこと

- 事実の創作・誇張（書かれていない感謝エピソードや病名を足す等）
- 過剰な謙譲・修飾で用件が埋もれた文
- 相手を非難する表現（苦情でも「事実 + 困っていること + お願い」の形に変換する）
- 媒体無視（LINE に「拝啓 盛夏の候」等）
