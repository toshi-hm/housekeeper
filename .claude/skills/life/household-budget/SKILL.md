# Skill: household-budget

家計の振り返りと、続けられる改善提案を作る。

## 原則

- **扱うのは金銭のプライベートデータ**。ユーザーが貼った内容の分析に徹し、
  外部送信・保存はしない。口座情報・カード番号などが貼られたら分析に不要と伝え、以後参照しない
- 投資・保険の個別商品推奨はしない（一般的な考え方の説明まで。具体判断は専門家へ）
- 説教しない。支出を裁かず、事実 → 選択肢の順で示す

## Step 1. データを受け取る

ユーザーに、ある形のままで貼ってもらう（整形を要求しない）:

- 家計簿アプリの月次サマリ / カード明細のコピペ / レシートの手打ちメモ、なんでも可
- 期間（1ヶ月分〜）と、分かれば手取り収入
- 任意: 貯蓄目標（「月N万貯めたい」「年内に◯◯資金」）

データが荒くても概算で進める。不足で分析が歪む場合のみ、その旨を明記する。

## Step 2. 分類・集計する

貼られたデータを標準カテゴリに整理する（ユーザーのカテゴリがあればそちらを優先）:

- **固定費**: 住居 / 通信 / 水道光熱 / 保険 / サブスク / 教育・保育
- **変動費**: 食費（内食・外食を分ける）/ 日用品 / 交通 / 医療 / 被服・美容 / 交際 / 趣味・娯楽
- **特別費**: 冠婚葬祭・家電購入・旅行など単発大型

分類に迷った項目は推測で割り当て、末尾に「分類の仮定」として明記する。

## Step 3. レポートを出す

```markdown
# 💰 家計レビュー（YYYY年M月）

## サマリ

収入 ¥--- / 支出 ¥--- / 収支 ±¥---（貯蓄率 --%）

## 支出の内訳

| カテゴリ | 金額 | 割合 | 目安との比較 |
| -------- | ---- | ---- | ------------ |

## 気づき（3〜5 個）

- 事実ベースで。例:「外食が食費の6割」「サブスク合計¥---/月（N契約）」

## 改善提案（インパクト × 手間 で優先順位付け）

| #   | 提案               | 月あたり効果 | 手間     | 種別   |
| --- | ------------------ | ------------ | -------- | ------ |
| 1   | （固定費から先に） | ¥---         | 一度だけ | 固定費 |
```

### 分析の質の基準

- **固定費 → 大型変動費 → 細かい変動費** の順で見る（効果と持続性の高い順）。
  「ラテマネーを我慢」系の消耗する提案を上位に置かない
- 目安比較（食費は手取りの15%前後 等）は「世帯構成で大きく変わる一般目安」と断って使う
- 効果は月額換算で示す（年額だけで大きく見せない）
- 提案は 3〜5 個まで。全部やれと言わず「まず #1 だけ」と勧める

## Step 4. フォローアップ

- 貯蓄目標があれば: 現状とのギャップ → どの提案の組み合わせで届くかを示す
- 「毎月やって」と言われたら、Claude Code の定期実行（Routine / スケジュール機能）で
  月次レビューを回す設定を提案する（次回は前月比の差分から入る）

## やってはいけないこと

- 特定の金融商品・保険・投資の推奨
- 「無駄遣いですね」等の評価的な物言い
- 少額の娯楽費を削る提案を最優先に置く（続かない）
- データにない支出の存在を断定する（「〜があるはず」ではなく「もしあれば」）
