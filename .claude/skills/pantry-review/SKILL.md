# Skill: pantry-review

家庭の在庫の健康診断レポートを作る。週次で回すことを想定した生活系スキル。
**読み取り専用**。書き込みは内容提示 + ユーザー承認を経た場合に限る。

## Step 0. 在庫データの入手経路を解決する

1. **同ディレクトリの `PROJECT.md` があれば読む** — 在庫データソース（DB 接続手段・クエリ）が
   定義されていればそれを使う
2. なければ完全汎用モード: ユーザーに以下を聞く（全部揃わなくてもあるだけで進める）:
   - いまの在庫（食品・日用品）と、分かる範囲の期限
   - 最近使った/使っていないものの感覚
   - 買い物リストや「切らしているもの」

## Step 1. 分析

### A. 期限アラート

- 期限切れ / 3日以内 / 7日以内 に分けて列挙
- 期限切れは「廃棄判断を」ではなく、まず **救済策**（加熱調理・冷凍・賞味期限なら状態確認）を添える
- 3〜7日のものは `recipe-from-stock` スキルでの献立化を提案する

### B. デッドストック

- 在庫があるのに長期間（目安 30 日）動きがないもの
- 「使い切りアイデア」or「次回から買わない候補」として提示

### C. 欠品・補充候補

- 使い切ったもののうち、消費頻度が高い定番品 → 切らしの検知
- 買い物リストに長期間残っているもの → 買い忘れ

### D. 記録衛生（在庫管理システムがある場合のみ）

- 期限未入力の食品（登録時の入力漏れの可能性）
- カテゴリ / 保管場所が未設定のアイテム
- 「実態は空のはず」のまま残っている記録

## Step 2. レポート出力

```markdown
# 🏠 在庫レビュー（YYYY-MM-DD）

## サマリ

在庫 N 品 / 🔴 期限切れ n / 🟠 3日以内 n / 💤 デッドストック n / 🛒 補充候補 n

## 🔴 今すぐ対応

（期限切れ・至急消費。1 行 1 アイテム + 救済策）

## 🟠 今週中に消費

（→ 献立化するなら recipe-from-stock を案内）

## 💤 眠っている在庫

（最終使用の目安 + 使い切りアイデア or 買わない提案）

## 🛒 買い物提案

（欠品定番品 + リスト積み残し。まとめて 1 回の買い物で済む形に）

## 🧹 記録のメンテ

（期限未入力・未分類など。在庫管理システムがある場合のみ）

## 前回からの変化

（前回レビューが会話履歴にあれば差分を 2〜3 行。なければ省略）
```

- 各セクション 0 件なら「なし ✅」と明記（空セクションを黙って消さない）
- 全体で問題ゼロなら褒めて短く終える。無理に指摘を作らない

## Step 3. フォローアップ提案

- 買い物提案の登録を望まれたら（在庫システム連携がある場合）、書き込み内容を
  提示 → 承認 → 実行の順で登録する
- 「毎週やって」と言われたら、Claude Code の定期実行（Routine / スケジュール機能）で
  週 1 回このスキルを起動する設定を提案する

## やってはいけないこと

- 承認なしの書き込み
- 「捨てましょう」の断定（廃棄判断はユーザーの領分。救済策を先に出す）
- 件数の多い在庫の全行列挙（30 品を超えたらカテゴリ集計 + 問題のあるものだけ列挙）
