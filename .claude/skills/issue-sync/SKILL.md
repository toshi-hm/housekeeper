# Skill: issue-sync

`PLANS.md` の §10 Phase 別 TODO を **マスター** とし、GitHub Issues と双方向同期する。

- 未起票タスク（`- [ ] ...` で行末に `<!-- issue:#NN -->` がない）→ `gh issue create` で起票し、行末に番号を書き戻す
- チェック済タスク（`- [x] ...`）で対応 Issue が open → `gh issue close`
- `PLANS.md` を「真実の源」として扱う。Issue 側のタイトル変更は同期対象外（必要なら手動で）

## 使う前提

- `gh` CLI が認証済み（`gh auth status` で確認）
- リポジトリに push 権限がある
- `PLANS.md` の §10 が以下の規約で書かれている:

```
- [ ] 未起票のタスク
- [ ] 起票済みのタスク <!-- issue:#42 -->
- [x] 完了したタスク <!-- issue:#42 -->
```

## 引数（ユーザー指示から判断）

| 指示の例                       | 動作                                                                 |
| ------------------------------ | -------------------------------------------------------------------- |
| 「dry run で」「確認だけ」     | `mode = dry-run`：差分を表示するだけで起票・close しない             |
| 「v1.2 だけ」「Phase v1 のみ」 | `phase = "v1.2"` などで Phase フィルタ（§10 配下の見出しと前方一致） |
| 何も指定なし                   | 全 Phase を対象に実際に同期する                                      |

## 実行手順

### Step 1. 前提確認

```bash
git rev-parse --is-inside-work-tree
gh auth status
```

どちらかが失敗したらエラー報告して停止。

### Step 2. PLANS.md をパース

`PLANS.md` を Read で取得し、`## §10 Phase 別 TODO` 以降を対象にする。
各タスク行を以下に変換する:

```ts
{
  phase: "v1" | "v1.1" | "v1.2" | "v1.3" | "Backlog",
  group: "1) 基盤 / 共通" 等の小見出し（任意）,
  title: "タスク文字列（チェックボックス・コメントを除く）",
  checked: boolean,
  issueNumber: number | null,
  lineNumber: number,
}
```

Phase は `### v1 — MVP（…）` のような H3 見出しから抽出する。
Group は H4 や `#### 1) 基盤 / 共通` のような小見出しから抽出する（無くてもよい）。

`phase` フィルタが指定されていれば、ここで絞り込む。

### Step 3. 未起票タスクを起票

対象: `issueNumber == null && checked == false`

各タスクごとに `gh issue create` を実行:

```bash
gh issue create \
  --title "<title>" \
  --body  "<body>" \
  --label "phase/<phase 小文字>" \
  --label "area/<group から推測, 不確実なら省略>"
```

- ラベルが未存在なら `gh label create phase/<phase> --color <色> --description "..."` で作成（dry-run 時は作成しない）
- 起票成功で返る URL から Issue 番号を抽出（例: `https://github.com/owner/repo/issues/42` → `42`）
- `PLANS.md` の該当行末尾に ` <!-- issue:#42 -->` を **追加** して Edit する
  - インデント・チェックボックス・前後の文脈を壊さないよう、行全体を書き換えるのではなく **末尾に追加するだけ**

#### Issue 本文テンプレート

```
## 関連
- PLANS.md: §10 Phase {{phase}} / {{group ?? "—"}}
- 元行: `{{title}}`

## 受け入れ条件（Definition of Done）
- [ ] 該当 Phase の意図に沿った実装が main にマージされている
- [ ] `bun run build` が通る
- [ ] `bun run lint` が通る
- [ ] `bun test` が通る（該当範囲のテストがある場合）
- [ ] 影響する spec（docs/specs/**）を更新した

## 参考 spec
- 必要に応じて `docs/specs/features/*.md` を参照すること

🤖 Generated by issue-sync skill
```

### Step 4. 完了タスクを close

対象: `checked == true && issueNumber != null`

```bash
gh issue view <NN> --json state -q .state
```

`OPEN` なら:

```bash
gh issue close <NN> --comment "PLANS.md でチェックされたため close"
```

### Step 5. レポート

ユーザーに以下のサマリを返す:

- **起票**: N 件（タイトルと URL の一覧）
- **close**: N 件（番号一覧）
- **スキップ**: N 件（既起票・既 close）
- **警告**: 記法ミスでパースに失敗した行（あれば）
- dry-run の場合は「実際には gh コマンドを実行していない」と明記

### Step 6. 後処理

- `PLANS.md` への書き戻しは行うが、**`git add` / `git commit` は行わない**（ユーザーが内容を確認してからコミットする）
- 同期結果のレポート末尾に「変更を確認後、`git add PLANS.md && git commit` してください」と添える

| label                                                                                                                                                                                      | 色             | 意味                 |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------- | -------------------- |
| `phase/v1`                                                                                                                                                                                 | `0e8a16`（緑） | MVP                  |
| `phase/v1.1`                                                                                                                                                                               | `fbca04`（黄） | 追加機能             |
| `phase/v1.2`                                                                                                                                                                               | `d93f0b`（橙） | PWA / 通知           |
| `phase/v1.3`                                                                                                                                                                               | `5319e7`（紫） | 統計                 |
| `phase/backlog`                                                                                                                                                                            | `bfbfbf`（灰） | 優先度未確定         |
| `area/auth` `area/inventory` `area/barcode` `area/expiry` `area/master-data` `area/storage` `area/shopping` `area/notifications` `area/pwa` `area/i18n` `area/stats` `area/db` `area/test` | 任意           | group の意味から推測 |

## 注意事項

- **未コミットの変更が他にある場合**は警告し `git status` を見せてから続行確認する
- 同じ文字列のタスクが複数 Phase にあっても **行番号で識別**するので衝突しない
- 起票後の Issue 番号書き戻しに失敗した場合（例: PLANS.md が他のプロセスで編集中）は、起票自体はキャンセルできないため **手動で書き戻す指示を report に出す**
- Codex などほかの AI でも同じ手順で実行できるよう、本ファイルだけで完結するように書いている
